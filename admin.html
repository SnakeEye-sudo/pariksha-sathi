Admin Access Locked Down

Client-side admin panel disabled

The old admin page exposed sensitive analytics access directly from the browser and included a hardcoded password pattern. That surface has been removed as part of the real hardening pass.

  • Normal student login still works with Google.
  • User data should now be protected by self-only Firestore rules.
  • Any future admin dashboard should use server-side verification, not client-side collection reads.
